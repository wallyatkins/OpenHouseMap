# project-context.md

## OpenHouseMap Project Context

### Technical Preferences
- **Frontend**: Vanilla HTML/CSS/JavaScript with Leaflet and Three.js
- **Storage**: localStorage for offline capability, GitHub for version control
- **Target**: Private property and home detail mapping tool
- **Architecture**: Client-side only, no backend required

### Implementation Rules
1. All data remains private on the owner's device
2. Blueprints and property data are user-provided
3. Map integration uses OpenStreetMap as baseline only
4. Versioning system tracks all property changes with provenance
5. Three.js for 3D visualization, Leaflet for 2D mapping

### Project Goals
- Property mapping with architectural-level detail
- House blueprint integration and 3D visualization
- Maintenance tracking and property management
- Private, owner-controlled data storage
