# BMAD Methodology — Open House Map

BMAD (Build → Measure → Analyze → Deploy) is an iterative development framework. Each cycle produces working software, collects data, learns from it, and improves.

## Phases

### Build
- Write code, add features
- Start small, work incrementally
- Each build should be a working, testable product
- **Current**: Basic map with features, 3D house model placeholder

### Measure
- Test what was built
- Collect feedback from actual use
- Document what works, what doesn't
- **Current**: Need to test with actual blueprint scans, measure accuracy

### Analyze
- Review measurements and feedback
- Identify gaps and improvements
- Prioritize next build cycle
- **Current**: Need to analyze blueprint integration approach

### Deploy
- Release to production environment
- Monitor for issues
- Collect real-world data
- **Current**: Local-first, deploy = save/export data

## Iteration Cycle

```
Build → Measure → Analyze → Deploy → (repeat)
```

Each cycle should:
1. Deliver incremental value
2. Collect measurable data
3. Generate actionable insights
4. Lead to next improvement

## Our BMAD Cycle for Open House Map

### Cycle 1: Foundation (Current)
**Build**: Map + basic features + 3D placeholder
**Measure**: User feedback on UI/UX, feature completeness
**Analyze**: What's missing for real property mapping?
**Deploy**: Local storage, JSON export

### Cycle 2: Blueprint Integration (Next)
**Build**: Blueprint upload + annotation + 3D integration
**Measure**: Test with actual house blueprints
**Analyze**: How well does 2D blueprint translate to 3D?
**Deploy**: Add to app, refine workflow

### Cycle 3: Detail Layers
**Build**: Full 3D house model with all layers
**Measure**: Test with actual house dimensions
**Analyze**: Is the level of detail useful for homeowners?
**Deploy**: Production-ready tool

### Cycle 4: Advanced Features
**Build**: Parcel boundary, utilities, maintenance tracking
**Measure**: Real-world usage patterns
**Analyze**: What features are actually used vs. unused?
**Deploy**: Final product

## Documentation Standards

Every phase produces:
- ✅ Working software
- ✅ Updated documentation
- ✅ Measured results
- ✅ Lessons learned
